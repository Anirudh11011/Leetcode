## 216. Combination Sum III

🔗 https://leetcode.com/problems/combination-sum-iii/

---

## 🧩 Problem

Find all valid combinations of `k` numbers that sum up to `n` such that the following conditions are true:

* Only numbers from `1` through `9` are used.
* Each number is used **at most once**.

Return a list of all possible valid combinations. The list must not contain the same combination twice, and the combinations may be returned in any order.

---

## 📌 Examples

### Example 1:

**Input:**

```python
k = 3, n = 7
```

**Output:**

```python
[[1,2,4]]
```

**Explanation:**

`1 + 2 + 4 = 7`

There are no other valid combinations.

---

### Example 2:

**Input:**

```python
k = 3, n = 9
```

**Output:**

```python
[[1,2,6],[1,3,5],[2,3,4]]
```

**Explanation:**

* `1 + 2 + 6 = 9`
* `1 + 3 + 5 = 9`
* `2 + 3 + 4 = 9`

There are no other valid combinations.

---

### Example 3:

**Input:**

```python
k = 4, n = 1
```

**Output:**

```python
[]
```

**Explanation:**

There are no valid combinations.

Using `4` different numbers in the range `[1,9]`, the smallest possible sum is:

`1 + 2 + 3 + 4 = 10`

Since `10 > 1`, no valid combination exists.

---

## 🔒 Constraints

* `2 <= k <= 9`
* `1 <= n <= 60`

---

## 💡 Approach (Backtracking)

* Use **backtracking** to generate combinations from numbers `1` to `9`.
* At each step:

  * Choose the current number.
  * Move to the next index because each number can be used only once.
* Add a combination to the result only when:

  * The sum becomes `n`
  * The length of the combination becomes `k`
* If the path length becomes `k` and the sum is not `n`, stop exploring that path.

---

## ⏱ Complexity

* **Time Complexity:** `O(2^9)` since each number from `1` to `9` can either be chosen or not
* **Space Complexity:** `O(k)` for recursion stack and current path

---

## 🧠 Solution (Python)

```python
class Solution:
    def combinationSum3(self, k: int, n: int) -> List[List[int]]:
        s = [1,2,3,4,5,6,7,8,9]

        def back(index, path, csum):
            if csum == n and len(path) == k:
                res.append(path[:])
                return

            if len(path) == k and csum != n:
                return

            for i in range(index, len(s)):
                path.append(s[i])
                csum += s[i]
                back(i + 1, path, csum)
                csum -= s[i]
                path.pop()

        res = []
        back(0, [], 0)
        return res
```
